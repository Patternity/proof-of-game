# Security Policy

## Scope

This repository contains **protocol specifications only**.
There is no production code, no deployed services, and no private infrastructure.

Nevertheless, security issues related to:
- consensus rules
- canonicalization ambiguity
- determinism violations
- protocol-level attack vectors

are taken seriously.

---

## Reporting a Vulnerability

If you discover a potential security issue or ambiguity that could affect
the correctness, safety, or consensus properties of the Proof-of-Game protocol:

**Please report it privately.**

Do NOT open a public issue.

---

## How to Report

At this stage, please contact the maintainers **directly via GitHub**.

Use one of the following methods:
- a private GitHub message to a repository maintainer, or
- a security advisory submission via GitHub (if enabled).

Public disclosure should be avoided until the issue is reviewed.

---

## Disclosure Policy

Valid reports will be:
- reviewed privately by the maintainers
- addressed in a future protocol revision if necessary
- acknowledged in release notes where appropriate

There are currently no bug bounties associated with this repository.

---

## Thank You

Responsible disclosure helps keep the protocol safe and deterministic.
